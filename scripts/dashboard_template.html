⚙ About TRIZ Engine

A Claude Code plugin that brings TRIZ methodology to AI-assisted problem solving

💡
40 Inventive Principles
Complete TRIZ knowledge base with all 40 inventive principles, contradiction matrix, and parameter mappings from Altshuller's original work.
⚖
Contradiction Analysis
Automatically identifies technical and physical contradictions in engineering problems and maps them to the 39x39 contradiction matrix.
🤖
3-Agent Pipeline
Contradiction Agent → Solution Agent → Evaluator Agent. Each specialized for its role, with LLM-as-judge scoring.
📊
8 MCP Tools
analyze_contradiction, suggest_principles, apply_principle, evaluate_solution, solve_with_triz, lookup_matrix, suggest_parameters, run_benchmark.
🎯
ARIZ-85C Algorithm
Full implementation of the Algorithm of Inventive Problem Solving, guiding users through structured contradiction resolution.
🏅
ELO Arena
Bradley-Terry ELO leaderboard comparing TRIZ-augmented solutions against vanilla LLM baselines across multiple benchmarks.

Quick Start

$ git clone https://github.com/sharathsphd/triz-engine.git
$ cd triz-engine
$ pip install -e ".[dev]"
# In Claude Code:
$ claude
> /triz:solve "How to increase battery capacity without increasing weight?"

Architecture

triz-engine/
├── commands/          # /triz:solve, /triz:benchmark, /triz:matrix
├── agents/            # contradiction-agent, solution-agent, evaluator-agent
├── skills/            # auto-triz activation skill
├── hooks/             # pre-prompt TRIZ detection hook
├── mcp/               # FastMCP server with 8 tools
├── benchmark/         # TRIZBENCH runner, scorer, ELO system
│   ├── problems/      # 12 canonical TB problems
│   └── external/      # MacGyver, CresOWLve, Patent adapters
├── knowledge/         # triz-knowledge-base.json, contradiction matrix
└── results/           # All benchmark output JSONs

📈 Methodology

How benchmarks are scored and ELO ratings computed

Internal TRIZBENCH Scoring (5 dimensions)
Contradiction Identification (CI) — 25% weight
Principle Selection (PS) — 20% weight
Solution Novelty (SN) — 20% weight, LLM-as-judge
Contradiction Resolution (CR) — 25% weight, LLM-as-judge
Ideal Final Result (IFR) — 10% weight
Final = weighted sum, max 100 points
External TRIZBENCH (Patent Analysis)
Evaluated against real patent contradiction data (236 patents):
Contradiction Type — exact match (20 pts)
Parameter Match — exact (30 pts) or partial (15 pts)
Principle F1 — precision/recall vs ground truth (50 pts)
Derived score = type + params + F1*50, max 100
MacGyver Benchmark
Constrained creative problem solving with limited objects.
Scored by LLM judge on a 4-level scale:
Perfect (1.0) — fully correct, creative
Good (0.75) — workable solution
Partial (0.5) — partially addresses problem
Fail (0.0) — doesn't solve the problem
CresOWLve Benchmark
Lateral/divergent creative reasoning riddles.
Binary scoring: correct or incorrect.
Answer matching uses normalized substring matching
with punctuation stripping and case-insensitive comparison.
ELO Rating System
Bradley-Terry model adapted for benchmark comparison:
• Initial rating: 1000 for each participant
• K-factor: 32 for first match, 16 thereafter (max of both players)
• Expected score: E = 1 / (1 + 10(Rb-Ra)/400)
• Win/Loss/Draw determined by comparing scores per problem
• 95% confidence intervals via 1000-iteration bootstrap
• Matches processed in order: Internal → External → MacGyver → CresOWLve
